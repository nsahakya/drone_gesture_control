# drone_gesture_control

Проект для управления дроном жестами рук через веб‑камеру.

Реализовано:
- распознавание руки и пальцев через **MediaPipe Hands**
- классификация простых жестов
- управление дроном **DJI Tello** через `djitellopy` (и режим `mock`, если дрона нет)

## Установка

```bash
cd /home/narek/Desktop/gesture_control
source .venv/bin/activate
pip install -r requirements.txt
```

Примечание: текущий `mediapipe` для Python 3.13 использует **Tasks API** и требует модель-бандл
`hand_landmarker.task`. Проект скачает её автоматически при первом запуске в `models/hand_landmarker.task`.

## Запуск

### Без дрона (проверка жестов)

```bash
python main.py --mode mock
```

### С дроном DJI Tello

1) Подключитесь к Wi‑Fi сети Tello.
2) Запустите:

```bash
python main.py --mode tello
```

## Жесты

Дискретные действия (нужно **удерживать** жест указанное время):
- `OPEN_PALM` (открытая ладонь) — удерживать ~1.2с, чтобы **включить/выключить ARM** (разрешить управление)
- `ROCK` (указательный + мизинец) — удерживать ~1.0с, чтобы **взлететь** (только когда ARM=true)
- `FIST` (кулак) — удерживать ~1.0с, чтобы **сесть** (только когда летит)

Непрерывное управление (работает только когда **ARM=true** и дрон в воздухе):
- `POINT` (указательный) — вперед
- `TWO` (указательный+средний) — назад
- `THREE` (указательный+средний+безымянный) — влево
- `FOUR` (4 пальца без большого) — вправо
- `THUMB_UP` — вверх
- `THUMB_DOWN` — вниз
- `PINKY` (мизинец) — поворот (yaw) влево
- `ROCK` (указательный + мизинец) — поворот (yaw) вправо
- `OPEN_PALM` или `NONE` — зависание (0 скорости)

## Управление и безопасность

- Нажмите `q` в окне видео, чтобы выйти (перед выходом код пытается посадить дрон, если он летит).
- Рекомендация: тестируйте сначала в `--mode mock`.
- Используйте открытое пространство и держитесь на безопасной дистанции.

## Параметры

```bash
python main.py --help
```

Полезные:
- `--camera 1` — если камера не `0`
- `--speed 60` — скорость (10..100)
- `--no-window` — запуск без окна OpenCV

## Если камера не работает

- Попробуйте другой индекс: `python main.py --mode mock --camera 1`
- Убедитесь, что устройство существует: `ls /dev/video*`
- Если вы в контейнере/WSL/удалённой сессии — камера может быть недоступна.
